# Overview
CRM(CDP) 사용을 위한 별도의 이전 작업 없이 기존의 스프레드시트를 기반으로 하여, 실시간으로 고객 데이터를 수집, 통합, 분석할 수 있는 경량화된 CDP(Customer Data Platform)를 개발합니다. 전자상거래(B2C) 기업에서 각 아이템에 '탄소 중립 점수'를 매겨 고객과 기업 모두 탄소 중립과 지속 가능성에 초점을 맞추도록 유도합니다.

# Core Features
- **데이터 수집**: Google Sheets 연동을 통한 실시간 데이터 수집, 향후 Excel, CSV, API 등 다양한 채널 지원 고려
- **데이터 정제 및 표준화**: 스프레드시트 열(Column) 매핑, 결측치/중복 제거, 타입 변환 등
- **고객 프로필 통합**: 여러 데이터 소스를 통합하여 단일 고객 프로필 구축
- **사용자 기반 수정 이력 관리**: 로그인한 ID로 식별, 변경 로그 기록 및 관리자 대시보드에서 조회
- **고객 세분화**: 조건 기반 세그먼트 생성, AI 기반 분석
- **AI 기반 분석**: 고객 이탈 예측, 구매 성향 예측, LTV 등
- **데이터 시각화**: 대시보드, 그래프, 차트 등 시각 자료 제공
- **데이터 내보내기 및 외부 연동**: 분석 결과를 Google Sheets로 내보내기, 향후 이메일 마케팅 툴 등과 연동 고려

# User Experience
- **주요 사용자**: 전자상거래(B2C) 기업, 데이터 관리에 익숙하지 않은 중소기업/스타트업 담당자
- **주요 플로우**: Google 로그인 → 스프레드시트 연동 → 데이터 업로드/동기화 → 데이터 정제/분석 → 대시보드/리포트 확인
- **UI/UX 고려사항**: 익숙한 시트 기반 인터페이스, 최소한의 클릭으로 데이터 업로드/동기화, 실시간 피드백, 직관적인 대시보드

# Technical Architecture
- **System Components**: Frontend(React), Backend(Node.js/Express), DB(MySQL), Google API 연동
- **Data Models**: 고객(customer), 시트(sheet), 분석 결과(analysis), 변경 로그(log)
- **APIs and Integrations**: Google Sheets API, Google OAuth, 내부 REST API
- **Infrastructure**: 클라우드 배포(예: AWS, GCP), 데이터 백업 및 보안, 확장성 고려

# Development Roadmap
- **MVP**: Google Sheets 연동, 고객 데이터 업로드/동기화, 데이터 정제/표준화, 기본 대시보드, Google OAuth 인증, 기본 분석(고객 수, 신규 가입, 구매 데이터 등)
- **Future Enhancements**: Excel/CSV 업로드, AI 기반 세분화/이탈 예측, 고도화된 대시보드, 외부 마케팅 툴 연동, 감사 로그/롤백, RBAC, 데이터 백업/복구

# Logical Dependency Chain
- Google OAuth/로그인 → Google Sheets 연동 → 데이터 업로드/동기화 → 데이터 정제/표준화 → 고객 프로필 통합 → 기본 분석/대시보드 → AI 분석/세분화 → 외부 연동/고도화
- 최대한 빠르게 usable한 프론트엔드(대시보드) 구현 후, 분석/AI/연동 기능을 점진적으로 추가

# Risks and Mitigations
- **기술적 난이도**: Google API 연동, 실시간 동기화 → 공식 문서/샘플 코드 적극 활용, MVP 범위 내 단순화
- **MVP 범위 산정**: 핵심 기능에 집중, 부가 기능은 후순위로 미룸
- **리소스 부족**: 오픈소스/외부 라이브러리 적극 활용, 자동화 도구 도입
