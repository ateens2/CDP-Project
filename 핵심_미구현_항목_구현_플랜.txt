# 5번 태스크(고객 프로필 통합) 핵심 미구현 항목 구현 플랜

## 1. DB 스키마 설계 및 마이그레이션
- 변경 이력(History) 테이블 설계 및 생성
  - 예시: customer_change_history (id, customer_id, field_name, old_value, new_value, changed_by, changed_at)
- 확장 속성 테이블(필요시)
- 기존 customer 테이블에 필요한 컬럼 추가 (created_at, updated_at 등)

## 2. 백엔드 서비스/로직 설계
- 변경 이력 기록 미들웨어/서비스 구현
- 프로필 병합/중복/충돌 해결 로직 구현
- 검색/페이징/정렬/필터링 API 구현

## 3. 프론트엔드 UI 설계
- 고객 상세/수정/이력/병합 화면 설계 및 구현

## 4. 테스트 및 데이터 마이그레이션
- 기존 데이터 마이그레이션
- 단위/통합 테스트 코드 작성
